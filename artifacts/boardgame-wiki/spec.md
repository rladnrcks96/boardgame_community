# 보드게임 위키 & 커뮤니티 — spec

## 개요

게임별 위키 문서(규칙 요약, 버전 이력)와 커뮤니티 기능(리뷰, 게시판, 댓글, 좋아요, 프로필)을 결합한 MVP. 비로그인 사용자는 게임 정보·위키·리뷰·게시글을 조회하고 검색할 수 있고, 로그인 사용자는 위키를 직접 편집하고 리뷰·게시글·댓글·좋아요로 참여할 수 있다.

## 범위

### 포함
- 이메일+비밀번호 회원가입, 이메일 인증, 로그인
- 메인 페이지: BGG 랭킹 기준 게임 그리드
- 게임 이름 검색
- 게임 상세 페이지: 메타데이터, 위키 본문, 편집 이력 목록, 평균 평점
- 위키 본문 편집 (로그인 필요, 승인 절차 없이 즉시 반영)
- 리뷰 작성/수정 (게임당 1인 1리뷰, 1~5점 정수, 고정 목록에서 다중 선택하는 태그)
- 게임 상세 페이지의 리뷰 태그 집계 표시
- 카테고리 게시판 (공략 / 변형룰 / 모임구인) 글 작성·조회
- 댓글 (단일 깊이)
- 좋아요 (게시글 대상)
- 프로필 페이지: 본인이 쓴 위키 편집 / 리뷰 / 게시글·댓글 목록, 획득한 업적 목록
- 업적 시스템: 5가지 최초 행동 기준 1회성 뱃지, 획득 즉시 토스트 알림 + 프로필 노출

### 제외
- 검색 상세 필터(인원수, 플레이타임 등) — 시딩 게임이 20-30개뿐이라 이름 검색만으로 충분, 게임 수가 늘어나면 재검토
- 위키 버전 간 diff 비교 화면 — 편집 이력 목록 노출로 우선 검증, 비교 UI는 필요성 확인 후 추가
- 포인트 누적·레벨·리더보드 — 누적 포인트 기반 보상은 저품질 스팸 행동을 유발할 수 있어 실사용 데이터 확보 후 규칙 결정. 1회성 최초 행동 업적(이번 spec에 포함)은 누적이 아니라 반복으로 얻을 수 없어 이 우려에 해당하지 않는다고 판단해 포함
- 리뷰 태그 자유 입력 — 고정 목록에서만 선택, 사용자가 새 태그를 만들 수는 없음
- 자체 평점으로 랭킹 소스 전환 — 자체 리뷰가 충분히 쌓이기 전까지 BGG 랭킹만 사용
- 대댓글(다중 깊이) — 단일 깊이로 모임구인/게시판 수요를 먼저 확인
- 모임구인 전용 필드(마감일/인원수) — 단순 게시판으로 시작, 수요 확인 후 구조화
- 다국어 지원 — 한국어 단일
- 룰북 원문 재게시 — 저작권 문제로 커뮤니티 재작성 방식만 허용

## 시나리오

### 1. 회원가입 및 이메일 인증
- **Given** — 회원가입 페이지, 이메일/비밀번호 입력
- **When** — "가입하기" 버튼 클릭
- **Then** — "인증 메일을 확인해주세요" 안내가 표시되고, 메일의 인증 링크를 클릭하기 전까지는 로그인이 되지 않는다

성공 기준:
- [ ] 신규 이메일 + 비밀번호 입력 → 가입 완료, 인증 메일 발송 안내 표시
- [ ] 이미 가입된 이메일로 가입 시도 → "이미 사용 중인 이메일입니다" 에러 표시
- [ ] 인증 링크 클릭 → "인증이 완료되었습니다" 표시, 이후 로그인 가능

### 2. 로그인
- **Given** — 이메일 인증을 완료한 계정
- **When** — 로그인 페이지에서 이메일/비밀번호 입력 후 "로그인" 클릭
- **Then** — 로그인되어 메인 페이지로 이동하고, 헤더에 로그인 상태가 표시된다

성공 기준:
- [ ] 인증된 계정 + 올바른 비밀번호 → 로그인 성공, 메인 페이지로 이동
- [ ] 인증되지 않은 계정으로 로그인 시도 → "이메일 인증이 필요합니다" 에러 표시, 로그인되지 않음
- [ ] 잘못된 비밀번호 → "이메일 또는 비밀번호가 올바르지 않습니다" 에러 표시

### 3. 메인 페이지 랭킹 그리드 조회
- **Given** — BGG 랭킹으로 시딩된 게임 목록이 존재하는 상태
- **When** — 비로그인 사용자가 메인 페이지에 접속
- **Then** — 게임 이미지, 이름, 평균 평점이 BGG 랭킹 순으로 그리드에 표시된다

성공 기준:
- [ ] 메인 페이지 접속 → 시딩된 게임 전체가 BGG 랭킹 순으로 카드 형태로 노출
- [ ] 아직 자체 리뷰가 없는 게임 → 평균 평점 대신 "리뷰 없음" 표시

### 4. 게임 검색
- **Given** — 검색창이 있는 화면
- **When** — 게임 이름의 일부를 입력
- **Then** — 이름이 일치하는 게임 목록이 표시된다

성공 기준:
- [ ] 존재하는 게임 이름 일부 입력 → 해당 게임(들)이 검색 결과에 표시
- [ ] 일치하는 게임이 없는 검색어 입력 → "검색 결과가 없습니다" 표시

### 5. 게임 상세 페이지 조회
- **Given** — 시딩된 게임 하나가 존재
- **When** — 비로그인 사용자가 해당 게임 상세 페이지에 접속
- **Then** — 게임 메타데이터(인원수/플레이타임/난이도/카테고리), 위키 본문, 편집 이력 목록(작성자·시각), 평균 평점, 리뷰 태그 집계가 표시된다

성공 기준:
- [ ] 게임 상세 페이지 접속 → 메타데이터와 위키 본문이 함께 표시
- [ ] 편집 이력이 있는 게임 → 편집자와 시각이 최신순으로 목록에 표시
- [ ] 편집 이력이 없는 게임(시딩 직후) → 이력 목록이 비어 있음을 나타내는 문구 표시
- [ ] 리뷰에 태그가 달린 게임 → "리뷰어 N명이 [태그]로 태그함" 형태로 태그별 집계가 표시
- [ ] 리뷰가 없는 게임 → 태그 집계 영역에 표시할 내용이 없음을 나타내는 문구 표시

### 6. 위키 본문 편집
- **Given** — 로그인한 사용자가 게임 상세 페이지에서 "편집" 버튼을 누른 상태
- **When** — 본문을 수정하고 "저장" 클릭
- **Then** — 수정된 본문이 즉시 게임 상세 페이지에 반영되고, 편집 이력에 새 항목이 추가된다

성공 기준:
- [ ] 로그인 사용자가 본문 수정 후 저장 → 상세 페이지에 수정된 본문이 바로 표시, 편집 이력에 항목 추가
- [ ] 비로그인 사용자가 "편집" 버튼 클릭 → 로그인 페이지로 이동
- [ ] 빈 본문으로 저장 시도 → "내용을 입력해주세요" 에러 표시, 저장되지 않음
- [ ] 두 사용자가 같은 문서를 거의 동시에 각자 수정해 저장 → 나중에 저장된 내용이 최종 본문이 되고, 편집 이력에는 두 저장 모두 기록됨

### 7. 리뷰 작성 및 수정
- **Given** — 로그인한 사용자가 게임 상세 페이지를 보는 상태
- **When** — 별점(1~5점), 리뷰 본문, 고정 목록에서 태그(0개 이상, 다중 선택 가능)를 선택 후 "등록" 클릭
- **Then** — 리뷰가 게임 상세 페이지의 리뷰 목록에 선택한 태그와 함께 추가되고, 평균 평점과 태그 집계가 갱신된다

성공 기준:
- [ ] 별점 4점 + 본문 입력 → 리뷰 목록에 추가, 평균 평점이 재계산되어 표시
- [ ] 태그 "마피아류", "파티게임" 선택 후 등록 → 리뷰에 두 태그가 함께 표시되고, 게임 상세의 태그 집계에 반영됨
- [ ] 태그를 하나도 선택하지 않고 등록 → 태그 없이 리뷰만 등록됨 (태그는 선택 사항)
- [ ] 고정 목록 밖의 값은 태그로 선택할 수 없음 (자유 입력 UI 자체가 없음)
- [ ] 이미 리뷰를 남긴 게임에 다시 리뷰 등록 시도 → 기존 리뷰(본문·별점·태그)가 수정되며 목록에 리뷰가 중복 생성되지 않음
- [ ] 비로그인 사용자가 리뷰 작성 시도 → 로그인 페이지로 이동

### 8. 게시판 글 작성 및 조회
- **Given** — 게임 상세 페이지의 게시판 탭
- **When** — 로그인 사용자가 카테고리(공략/변형룰/모임구인) 선택 후 제목/본문 입력, "등록" 클릭
- **Then** — 해당 카테고리 게시판 목록에 새 글이 나타난다

성공 기준:
- [ ] 카테고리 선택 + 제목/본문 입력 → 등록 즉시 해당 카테고리 목록 최상단에 표시
- [ ] 제목 없이 등록 시도 → "제목을 입력해주세요" 에러 표시
- [ ] 비로그인 사용자가 목록 조회 → 글 목록과 본문을 볼 수 있음 (읽기는 로그인 불필요)
- [ ] 비로그인 사용자가 "글쓰기" 클릭 → 로그인 페이지로 이동

### 9. 댓글 작성
- **Given** — 로그인 사용자가 게시글 상세 화면을 보는 상태
- **When** — 댓글 입력 후 "등록" 클릭
- **Then** — 댓글이 해당 게시글 아래 목록에 시간순으로 추가된다

성공 기준:
- [ ] 댓글 입력 후 등록 → 게시글 하단 댓글 목록에 즉시 추가
- [ ] 빈 댓글 등록 시도 → 등록되지 않고 에러 표시
- [ ] 비로그인 사용자가 댓글 입력 시도 → 로그인 페이지로 이동

### 10. 게시글 좋아요
- **Given** — 로그인 사용자가 좋아요를 누르지 않은 게시글을 보는 상태
- **When** — "좋아요" 버튼 클릭
- **Then** — 좋아요 수가 1 증가하고 버튼이 눌린 상태로 바뀐다

성공 기준:
- [ ] 좋아요 클릭 → 좋아요 수 +1, 버튼이 활성 상태로 표시
- [ ] 이미 누른 좋아요를 다시 클릭 → 좋아요 취소, 수가 -1
- [ ] 비로그인 사용자가 좋아요 클릭 → 로그인 페이지로 이동

### 11. 프로필 페이지 조회
- **Given** — 로그인한 사용자
- **When** — 본인 프로필 페이지에 접속
- **Then** — 본인이 편집한 위키 문서 목록, 작성한 리뷰 목록, 작성한 게시글·댓글 목록, 획득한 업적 뱃지가 각각 표시된다

성공 기준:
- [ ] 위키를 편집한 적 있는 사용자 → 프로필에 편집한 게임 문서 목록이 표시
- [ ] 리뷰를 남긴 적 있는 사용자 → 프로필에 작성한 리뷰 목록이 표시
- [ ] 업적을 획득한 사용자 → 프로필에 획득한 뱃지가 표시
- [ ] 아무 활동도 없는 신규 사용자 → 각 목록에 "아직 활동이 없습니다" 문구 표시

### 12. 업적 획득
- **Given** — 로그인한 사용자가 아직 특정 업적을 획득하지 않은 상태
- **When** — 해당 업적의 조건에 해당하는 첫 행동을 완료 (첫 게시글 작성 / 첫 리뷰 작성 / 첫 위키 편집 / 본인 게시글이 첫 좋아요를 받음 / 첫 댓글 작성)
- **Then** — 화면에 업적 획득 토스트 알림이 표시되고, 해당 뱃지가 프로필 페이지에 나타난다

성공 기준:
- [ ] 첫 게시글 작성 → "첫 게시글 작성" 업적 토스트 표시, 프로필에 뱃지 추가
- [ ] 첫 리뷰 작성 → "첫 리뷰 작성" 업적 토스트 표시, 프로필에 뱃지 추가
- [ ] 첫 위키 편집 → "첫 위키 편집" 업적 토스트 표시, 프로필에 뱃지 추가
- [ ] 본인이 쓴 게시글이 처음으로 좋아요를 받음 → 게시글 작성자에게 "첫 좋아요 받음" 업적 토스트 표시, 프로필에 뱃지 추가 (좋아요를 누른 사람에게는 표시되지 않음)
- [ ] 첫 댓글 작성 → "첫 댓글 작성" 업적 토스트 표시, 프로필에 뱃지 추가
- [ ] 이미 획득한 업적의 조건에 해당하는 행동을 다시 함 (예: 두 번째 게시글 작성) → 토스트가 다시 표시되지 않고, 뱃지도 중복 추가되지 않음

## 불변 규칙

- **권한**: 비로그인 사용자는 어떤 화면에서도 위키 편집, 리뷰 작성, 게시글/댓글 작성, 좋아요를 실행할 수 없다 — 시도 시 예외 없이 로그인 페이지로 이동한다
- **데이터 일관성**: 화면에 표시되는 게임 평균 평점은 등록된 리뷰들의 평균과 항상 일치한다
- **데이터 일관성**: 화면에 표시되는 게시글 좋아요 수는 실제 좋아요 기록 수와 항상 일치한다

## 의존성

- BGG 랭킹 기준으로 시딩된 20-30개 게임의 메타데이터(이름/이미지/인원수/플레이타임/난이도/카테고리)가 존재해야 한다
- 이메일 발송이 가능한 인증 시스템이 구성되어 있어야 한다

## 미결정 항목

없음 — 확인이 필요했던 항목은 모두 사용자와 논의해 결정했다.
